## 1. Where the 2026-06-20 baseline is wrong (or lying)

The first baseline (`reports/baseline-2026-06-20.json`) ranks sources by
mean `quality_score`. Three failure modes that are not yet captured:

1. **Binary licence labels overstate licence-cleanliness.** The Wikimedia
   adapter's `LicenseShortName` regex collapses "CC-BY-3.0 with
   non-portable derivatives clause", "PD-USGov", and "Cc-by" all to the
   same enum. The downstream training pipeline cannot tell the difference
   between a high-confidence `CC_BY` and a string-matched one.
   *Adversarial bet:* >10 % of Commons cooking videos have ambiguous
   licence text where a human would not be confident in the label.
2. **No eval-set contamination guard.** When we benchmark against
   YouCook2 / EPIC-KITCHENS / HD-EPIC we will silently leak training data
   into the eval set. Today there is no blocklist primitive at all.
3. **The comparison harness compares sources, but never compares
   *filters*.** AGENTS.md's Comparison-First Rule says "if you cannot
   beat the current baseline … document why the experiment was still
   informative." We cannot honour this rule without an *ablation*
   harness that toggles individual quality components and prints the
   delta. Right now adding a new quality heuristic just silently
   re-weights the score.
4. **English-only seed terms.** §3 of the prior plan flagged this and we
   shipped it anyway. The substitute-source graph (Wikimedia, IA,
   PeerTube) is *more* skewed toward English/Western than YouTube, so
   this is a quietly compounding bias.
5. **No "procedural density" signal.** A 4K plated-food slideshow scores
   the same as a shaky knife-on-onion demo. World-model utility
   correlates with procedural density (verbs / step-text / temporal
   ordering), not just resolution + duration.

## 2. Hypotheses for 2026-06-24

> **H2.** Adding a `license_confidence ∈ [0,1]` field per adapter and
> treating `license_clean := is_redistributable × confidence` strictly
> *lowers* mean quality on the 2026-06-20 fixture set (because some
> formerly "clean" records get partial credit), but the resulting
> ranking is more faithful to what a human reviewer would accept.
> Falsifiable: the relative ordering of sources is allowed to change;
> Wikimedia must remain ≥ Archive.org on `confidence × cleanliness`.

> **H3.** A `procedural_density` quality component built only from
> metadata (recipe-step count, imperative-verb density in title +
> description, presence of numbered steps) predicts whether a clip is
> "hands + objects + state change" without any video bytes. We will
> verify on a small hand-labelled subset of the existing fixtures.

> **H4.** A pHash + title-normalisation **blocklist** drawn from any
> public eval set (we ship a YouCook2-shaped fixture) is sufficient to
> reject the obvious overlaps. Recall on the synthetic overlap set must
> be ≥ 0.95 with zero false positives on the existing
> `tests/fixtures/` corpus.

> **H5.** A *systematic ablation* over quality components — running the
> comparison harness with each component independently zeroed —
> identifies the single highest-value component for the cooking
> vertical. Whichever wins, *that* component is the one to harden next
> session.

## 3. Concrete deliverables for `Coding-Agent` (today)

- [ ] `src/specint/records.py` — add `license_confidence: float = 1.0`
      to `VideoRecord` (default 1.0 keeps existing tests valid).
- [ ] Per-adapter confidence emission:
  - Wikimedia: 1.0 for CC0 / PD; 0.9 for clean CC-BY / CC-BY-SA;
    0.6 if `LicenseShortName` matched only by substring fallback;
    0.0 for `UNKNOWN`.
  - Internet Archive: 1.0 if `licenseurl` host is `creativecommons.org`;
    0.7 if license inferred from item-level metadata; 0.0 for unknown.
  - PeerTube: 0.95 if `licence.id ∈ {1,2,7}`; 0.0 otherwise.
  - Common Crawl: hard-coded 0.2 — the page rarely declares a CC licence
    in the JSON-LD block and we will *not* trust the embedded
    `VideoObject.license` field.
- [ ] `src/specint/contamination.py` — pure-Python blocklist:
  - JSONL on disk: `{kind: "title_norm" | "url" | "native_id" |
    "phash16", value: "..."}`.
  - `normalize_title()` → lowercase, NFKD-strip diacritics, collapse
    whitespace, drop punctuation, drop stop-words.
  - `Blocklist.load(path) -> Blocklist`,
    `Blocklist.contains(record) -> bool`,
    `filter_contaminated(records, blocklist) -> records`.
  - Ship a fixture `tests/fixtures/blocklists/youcook2_sample.jsonl` and
    a test that an obvious overlap is caught and an unrelated record is
    not.
- [ ] `src/specint/quality/metrics.py` — add
  `procedural_density` component:
  - 0.5 × min(1, num_recipe_steps / 6)
  - 0.3 × min(1, imperative_verb_count / 5)
  - 0.2 × (1 if title contains a digit-anchored step marker else 0)
  - Reweight existing components proportionally so total weight = 1.0.
- [ ] `src/specint/compare/ablation.py` — for a given source bundle,
  re-run the harness with each quality component set to zero in turn;
  emit a `reports/ablation-YYYY-MM-DD.json` with per-component
  *delta* against the all-on baseline.
- [ ] `scripts/precommit_forbid_youtube.py` — exit non-zero if any
  tracked file outside `docs/` contains a `youtube.com` or `youtu.be`
  URL; wired into `.pre-commit-config.yaml` and the CI `precommit`
  job. (Mirrors AGENTS.md hard-constraint #1.)
- [ ] `scripts/manifest_lint.py` — schema-checks every `*.jsonl` under
  `data/manifests/` (when present) against `VideoRecord`. CI job
  `manifest-lint` runs it.
- [ ] `scripts/pipeline_dry_run.py` — feeds the four checked-in
  fixtures through `parse → score → contamination.filter → compare`,
  writes a manifest under `data/manifests/dry-run-YYYY-MM-DD.jsonl`,
  and asserts byte-for-byte determinism across two consecutive runs.
  Wired into `tests/test_e2e_pipeline.py`.
- [ ] `reports/baseline-2026-06-24.json` regenerated through the new
  scorer so future runs have something to beat.

## 4. Out of scope (still)

- Live network calls in unit tests.
- Any source requiring auth or paid keys (Vimeo CC, YouTube CC, etc. —
  the YouTube-CC API endpoint is *legal* but explicitly excluded by
  AGENTS.md hard-constraint #1 because we cannot acquire media bytes
  without DRM-circumvention exposure).
- Distributed crawling, byte-level pHash on video, learned filters
  (planned for the next adversarial cycle).

## 5. Risks the next reviewer should challenge

- `procedural_density` is metadata-only and we have no
  ground-truth labels. A learned filter (planned v1, `quality/learned/`)
  is the right way to validate; the heuristic here is a *baseline*.
- The blocklist `phash16` kind has no implementation today — it is a
  schema placeholder for the byte-level pHash module that lands once
  we begin frame extraction. Title-normalised matching is the only
  active detector this session.
- Confidence weighting introduces a *new* free parameter
  (`confidence_threshold`); the ablation harness must include the
  no-threshold case so we do not silently launder bad heuristics
  through the new metric.
