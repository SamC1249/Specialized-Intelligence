### §7.4 — Eval-set contamination blocklist (`src/specint/contamination.py`)
- `Blocklist` + `BlocklistEntry` with JSONL round-trip and a default
  Hamming-8 radius (matching D-2026-06-20-4).
- `Blocklist.reject(candidates)` returns the offending pairs so the
  caller can audit, not just drop.
- Wired into the pipeline dry-run as a `contamination_gate` stage between
  `dedup` and `curate`.
- Seed blocklist at `data/blocklists/eval_seed.jsonl` (placeholders;
  populated by a future `scripts/build_eval_blocklist.py`).
- Unit tests: hex validation, radius semantics, empty blocklist returns
  distance 64, JSONL comment lines tolerated, round-trip stable.

### §7.6 — Per-record license-confidence (`src/specint/license_confidence.py`)
- Coarse heuristic in [0, 1] that combines: declared-license tier,
  license-URL pattern match against creativecommons.org / rightsstatements.org,
  contradiction phrases in title/description ("non-commercial",
  "all rights reserved", etc.), and a "known-clean source" bonus.
- `License.UNKNOWN` and `License.RESTRICTED` are always 0.
- `filter_by_confidence(records, threshold=...)` is the curator-facing API.

### §7.3 — Cuisine / language / corpus-bias audit (`src/specint/audit/`)
- `audit_records(records) -> AuditReport.to_dict()` emits a deterministic,
  JSON-serialisable report with cuisine counts, language counts, source
  counts, license counts, and per-cuisine duration totals.
- `classify_cuisine(text)` and `detect_language_iso2(text)` are pure
  keyword classifiers — the *systematic* signal that lands in CI
  artifacts. Not a substitute for a sampled human audit (called out in
  the module docstring).

### §7.2 — World-model utility proxy (`src/specint/wm_utility.py`)
- `wm_utility_score(record, motion_score) -> float in [0, 1]` weighting
  motion, recipe-step presence, resolution, duration sweet spot, and an
  anti-static-montage factor (penalises ASMR / beauty-shot / macro
  keywords when motion is low). Weights live in `DEFAULT_WEIGHTS` and are
  overridable so a future learned filter can drop in behind the same
  interface.
- `rank_records(records, motion_scores=...)` returns the records ordered
  by utility, deterministic-tie-broken on record id.

### Systematic strategy comparison (`src/specint/compare/strategies.py`)
- `compare_strategies(records, k, seed)` runs five rankers on the same
  candidate pool — `quality_metadata`, `wm_utility`, `license_confidence`,
  `license_then_wm`, and a seeded `random_baseline` — and emits a
  deterministic `StrategyResult` per strategy with mean quality, mean
  WM-utility, mean license-confidence, license-clean ratio, unique
  authors, and total duration.
- `strategy_overlap_matrix(results)` is the pairwise Jaccard table over
  selected_ids; the diagonal is 1.0 and the matrix is symmetric.
- This is the answer to "compare strategies systematically, not in
  one-off moments": every CI run regenerates the table and the diff is
  reviewable.

### CLI additions (`python -m specint`)
- `specint audit  --output …` writes the bias report to `reports/`.
- `specint strategies --top-k … --output …` writes the strategy
  comparison + overlap matrix to `reports/`.
- Both consume the existing offline fixtures (Wikimedia, Internet Archive,
  PeerTube, Common Crawl). Network is not touched.

## Open questions for the next adversarial review

1. Should the contamination gate use BK-tree indexing instead of linear
   scan once the blocklist crosses ~50k entries? Linear is fine at v0.
2. The bias-audit cuisine vocabulary is overwhelmingly Western — by
   construction. The next round should propose a multilingual expansion
   sourced from Wikidata cuisine taxonomy (`Q1778821` and descendants).
3. Strategy comparison currently uses metadata-only proxies for both
   the ranker and the evaluation. We need a *held-out, license-clean*
   evaluation set so the overlap table has a ground-truth column.
4. `license_confidence` does not yet read the upstream extmetadata.
   `LicenseShortName`-conflict signal directly; once Wikimedia adapter
   exposes the raw blob, we can derive a higher-confidence per-record
   contradiction flag.
