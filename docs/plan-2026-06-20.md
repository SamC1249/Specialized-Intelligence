# Adversarial-Agent Plan — 2026-06-20

> Note: this seed plan was authored together with the bootstrap commit
> because no prior `docs/plan-*.md` existed. Future plans should be written
> by `Adversarial-Agent` before `Coding-Agent` runs.

## Hypothesis under test

> **H1.** For frontier video world-modeling, *legally permissive* cooking
> footage on the open web is dominated by three sources — Wikimedia
> Commons, Internet Archive, and the recipe long-tail discoverable through
> Common Crawl `schema.org/VideoObject` blocks — and a quality-weighted
> blend of all three Pareto-dominates any single source on
> (yield × license-cleanliness × resolution × procedural-density).

## Why this is a frontier problem

Cooking video is *the* canonical "long-horizon, multi-object,
intent-driven, partially-observable" domain. Hands occlude objects, state
changes are irreversible (you can't un-fry an egg), and language↔action
grounding is unusually dense ("simmer until reduced by half"). This makes
it an ideal stress test for world models, *but* the obvious data source
(YouTube cooking channels) is mostly all-rights-reserved and therefore off
limits under our hard constraints.

## Adversarial questions to attack today

1. **Yield.** How many *unique, license-clean* hours can each source
   plausibly deliver per crawl-day? Estimate from listing/search APIs
   without downloading media.
2. **Provenance integrity.** Can we always recover (license, source URL,
   author) for a record? Build the canonical schema *before* writing any
   scraper.
3. **Quality-vs-quantity Pareto.** What heuristics can we compute from
   *metadata only* (no video download) that predict downstream usefulness?
   Initial candidates: declared duration, declared resolution, presence of
   recipe/step text, language detection on title+description, license
   tier.
4. **Comparability.** If two scrapers report "I found N hours," can we
   diff them deterministically? Need a `BenchmarkResult` schema and a
   `compare` CLI today, not later.
5. **Test discipline.** Network-flakiness will silently rot our pipeline.
   All tests must be offline; live calls live behind an env-gated
   integration suite.

## Concrete deliverables for `Coding-Agent`

- [ ] `src/specint/records.py` — `VideoRecord`, `License`, `SourceQuery`
      typed dataclasses (Pydantic v2). This is the single source of truth
      and must match `db_structured.md`.
- [ ] `src/specint/sources/base.py` — `BaseSource` ABC with
      `search(query) -> Iterable[VideoRecord]` and a pure-function
      `parse(raw: dict) -> VideoRecord` so we can unit-test against fixtures.
- [ ] Adapters (offline-parseable, fixtures in `tests/fixtures/`):
  - [ ] `wikimedia.py` — Commons MediaWiki API JSON
  - [ ] `archive_org.py` — Internet Archive advancedsearch.php JSON
  - [ ] `peertube.py` — PeerTube `/api/v1/search/videos` JSON
  - [ ] `common_crawl.py` — parse `schema.org/VideoObject` JSON-LD blocks
        out of an HTML fixture
- [ ] `src/specint/quality/metrics.py` — pure metadata-only quality
      scoring. Document each heuristic.
- [ ] `src/specint/compare/harness.py` — runs every registered source
      against the same `SourceQuery`, computes per-source aggregates,
      emits a JSON `BenchmarkResult` to `reports/`.
- [ ] `src/specint/cli.py` — `python -m specint compare`,
      `python -m specint sources`.
- [ ] Tests: one unit test per adapter using a checked-in fixture; one
      end-to-end test that runs the comparison harness across all
      adapters using fixtures (no network).
- [ ] CI: GitHub Actions matrix on Python 3.11 / 3.12 running
      `pre-commit` + `pytest`.
- [ ] Pre-commit: ruff (lint+format), mypy (best-effort), pytest-quick.

## Out of scope today

- Actual video frame extraction (we are URL+metadata only).
- Any source requiring auth or paid keys.
- Distributed crawling infra (single-process is fine until we have a
  baseline).

## Risks the next reviewer should challenge

- License classification is heuristic; we may mislabel a Commons video
  uploaded under a non-redistributable license. Mitigation: default to
  `License.UNKNOWN` and exclude from training corpora.
- Common Crawl JSON-LD blocks frequently lie about `contentUrl`. Always
  treat the *page* license, not the embedded video provider's license.
- "Cooking" search terms vary by language. Today we ship English seed
  terms only; multilingual is an explicit follow-up.
